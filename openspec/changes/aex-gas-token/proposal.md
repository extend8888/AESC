# 变更提案：AEX Gas 代币（链层完整实现）

## 状态：进行中

> **说明**：本变更提案涵盖 AEX 链层系统的完整功能实现。
> AEX 与 AESC 是两个独立系统，先完成 AEX 链层，再实现 AESC 合约层。

---

## 1. 概述

将 AESC 链的 Gas 代币从 `usei` 改为 `uaex`，并实现完整的 AEX 经济模型，包括：
- 基础配置（已完成）
- 通胀机制
- 手续费销毁与动态调节
- 净供给硬约束
- 动态 Gas 价格

## 2. 动机

AESC 链 fork 自 Sei Chain，需要：
1. 将默认的 `usei` 代币替换为 `uaex` (AEX) 作为 Gas 代币
2. 实现 AEX 完整经济模型，确保代币供给可控、激励合理

## 3. 代币参数

| 参数 | 值 |
|------|-----|
| 名称 | AEX |
| 符号 | AEX |
| 最小单位 | uaex |
| 精度 | 6 (1 AEX = 10^6 uaex) |
| 初始发行量 | 500,000,000 AEX |
| 用途 | 交易手续费（Gas） |
| 地址前缀 | aesc |

### 4.2 AEX-P1：通胀与供给控制 ⏳ 待实现

| 工作项 | 说明 |
|--------|------|
| 通胀机制 | 年通胀上限 3%，基于链上指标触发 |
| 通胀触发条件 | 交易量、区块稳定性、Gas 使用率 |
| 净供给硬约束 | 任意 12 个月净增发 ≤ 初始量 5% |

**技术实现：**

- 使用 `x/epoch` 的 `AfterEpochEnd` hook
- 新增状态存储记录 12 个月滚动窗口数据

### 4.3 AEX-P2：手续费销毁与动态调节 ⏳ 待实现

| 工作项 | 说明 |

| 手续费销毁 | 部分 fee 发送到 burn 模块账户 |
| 动态销毁比例 | 30% - 60%，基于 Gas 使用率自动调节 |
| 验证者分配 | 70% - 40%，剩余部分 |

- 修改 `x/evm/ante` 中的 fee 处理逻辑
- 在 EndBlocker 中执行销毁
- 与 `x/mint` 模块联动计算净供给
