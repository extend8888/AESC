# 变更：将 Sei 品牌标识替换为 AESC

## 为什么

AESC Chain 是基于 Sei 分叉的独立区块链项目，需要建立自己的品牌标识。当前代码库中仍保留大量 Sei 的标识（如账户前缀 `sei`、代币面额 `usei`、地址前缀 `sei1` 等），这些需要系统性地替换为 AESC 的品牌标识，以确保：

1. **品牌一致性**：所有用户可见的标识符都应该反映 AESC 品牌
2. **技术独立性**：建立独立的技术身份，避免与 Sei 混淆
3. **测试完整性**：确保所有测试使用正确的 AESC 标识符
4. **文档准确性**：技术文档应该放在正确的位置并使用正确的术语

虽然 `app/params/config.go` 已经完成了基础配置的修改，但代码库中仍有约 177 个文件包含 `usei` 引用，需要系统性地进行替换和测试。

## 变更内容

### 1. 核心标识符替换
- 代币面额：`usei` → `uaex`（微单位），`sei` → `aex`（标准单位）
- 地址前缀：`sei` → `aesc`（Bech32 前缀）
- 地址示例：`sei1...` → `aesc1...`（所有测试和示例地址）
- 链名称：在适当的地方从 `sei` 更新为 `aesc`

### 2. 代码文件更新
- 核心模块代码（`x/evm`, `x/mint`, `x/oracle` 等）
- 测试文件（单元测试、集成测试）
- 示例代码和合约
- 工具和脚本

### 3. 配置文件更新
- Genesis 配置模板
- 部署脚本（`poc-deploy/`, `depoly-scripts/`）
- Docker 配置
- 快速启动脚本

### 4. 开发工具增强
- **新增** Makefile `test` 命令，方便运行测试
- **新增** Makefile `test-unit` 命令，只运行单元测试
- **新增** Makefile `test-integration` 命令，只运行集成测试

### 5. 文档整理
- 将 `tmp/AESC 链开发技术方案.md` 移动到 `openspec/changes/rebrand-sei-to-aesc/design.md`
- 更新文档中的所有 Sei 引用

### 6. 验证和测试
- 运行所有单元测试确保替换正确
- 运行集成测试验证端到端功能
- 验证 genesis 配置可以正确初始化链

## 影响

### 受影响的能力
- `chain-identity`：链的基础标识和配置
- `token-economics`：代币面额和经济模型
- `address-format`：地址格式和前缀
- `testing-infrastructure`：测试基础设施
- `deployment-tools`：部署和运维工具

### 受影响的代码范围
- **核心配置**：`app/params/config.go`（已完成）
- **EVM 模块**：`x/evm/` 下的所有文件（约 20+ 文件）
- **其他模块**：`x/mint/`, `x/oracle/`, `x/tokenfactory/` 等
- **测试文件**：约 150+ 测试文件
- **部署脚本**：`poc-deploy/`, `depoly-scripts/`, `docker/`
- **构建系统**：`Makefile`
- **文档**：`tmp/` 目录下的技术文档

### 破坏性变更
- **地址格式变更**：所有现有的 `sei1...` 地址将无法使用，需要重新生成为 `aesc1...` 格式
- **代币面额变更**：所有使用 `usei` 的配置和脚本需要更新为 `uaex`
- **Genesis 配置**：需要使用新的 genesis 模板重新初始化链

### 迁移路径
由于这是一个新链的品牌重塑，不存在现有用户数据迁移问题。所有开发和测试环境需要：
1. 清理旧的链数据
2. 使用新的 genesis 配置重新初始化
3. 重新生成所有测试账户和密钥

## 依赖关系
- 依赖于 `aex-gas-token` 提案中定义的代币经济模型
- 为后续的 `aesc-staking-contracts` 提案提供正确的基础标识

## 风险和缓解措施

### 风险 1：遗漏某些文件中的引用
**缓解**：使用自动化脚本搜索所有可能的引用，并在测试中验证

### 风险 2：测试失败
**缓解**：分阶段进行替换，每个阶段运行测试验证

### 风险 3：配置文件格式错误
**缓解**：使用现有的配置文件作为模板，仔细验证 JSON 格式
